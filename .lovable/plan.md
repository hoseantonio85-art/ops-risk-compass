
# Ревью соответствия цветов эталонной палитре

## Что не так сейчас

Сравнил текущие токены в `src/index.css` и хардкод-цвета в компонентах с эталонной палитрой из скриншота. Палитра построена на 8 семействах × 10 ступеней (50–900): **Gray, Red, Amber/Orange, Green, Blue, Purple, Pink, Deep Orange** — все с конкретными HEX. У нас же часть значений «на глазок» и есть откровенный хардкод в виджетах.

### 1. Серые (фон, поверхности, бордеры) — оттенок и светлота не совпадают
Эталон Gray: `#F9FBFC → #F2F5F7 → #E1E4E8 → #D1D5DA → #959DA5 → #6A737D → #586069 → #444D56 → #2F363D → #24292E` (холодный сине-серый, **HSL ≈ 210 14% L**).

У нас сейчас (`220 16% …`):
- `--background: 220 16% 96%` → должен быть `gray-100 #F2F5F7` ≈ `210 14% 96%`
- `--card: 0 0% 100%` → ок (белый)
- `--secondary / --muted: 220 16% 94%` → должен быть `gray-100/200`, ближе к `210 14% 95%`
- `--border: 220 16% 90%` → `gray-200 #E1E4E8` ≈ `213 10% 89%`
- `--border-strong: 220 16% 84%` → `gray-300 #D1D5DA` ≈ `213 8% 84%`
- `--muted-foreground: 220 10% 46%` → `gray-500 #6A737D` ≈ `211 8% 45%`
- `--foreground: 222 25% 12%` → `gray-900 #24292E` ≈ `213 9% 16%` (у нас слишком насыщенный и тёмный)
- `--neutral: 220 12% 60%` → `gray-400 #959DA5` ≈ `213 7% 62%`
- `--neutral-soft: 220 16% 92%` → `gray-200` ≈ `213 10% 89%`

Эффект: фон/тексты сейчас отдают чуть синевой относительно эталона.

### 2. Бренд (primary) — не из палитры
Эталон Green: `#ECFDF3 → #D1FADF → #A6F4C5 → #6CE9A6 → #32D583 → #12B76A → #039855 → #027A48 → #05603A → #054F31`.

Сейчас `--primary: 158 70% 42%` — это близко к **green-600 `#039855`** (`152 96% 30%`), но насыщенность и тон не совпадают. Правильно:
- `--primary` = `green-600 #039855` ≈ `152 96% 30%` (или `green-500 #12B76A` ≈ `152 82% 39%` — выбрать одну ступень)
- `--primary-soft` (`158 70% 94%`) → `green-50 #ECFDF3` ≈ `152 81% 96%`
- `--accent / --accent-foreground` → `green-50` / `green-700 #027A48`
- `--ring` → совпадает с primary
- `--calm`, `--calm-soft`, `--calm-foreground` (158 64% 42% / 60% 94%) → green-600 / green-50

### 3. Семантика «attention» (амбер)
Эталон Amber: `#FFFAEB → #FEF0C7 → #FEDF89 → #FEC84B → #FDB022 → #F79009 → #DC6803 → #B54708 → #93370D → #7A2E0E`.

Сейчас:
- `--attention: 38 92% 52%` → близко к **amber-500 `#F79009`** (`32 95% 50%`), но смещено в жёлтый. Правильно `32 95% 50%`.
- `--attention-soft: 38 100% 94%` → `amber-50 #FFFAEB` ≈ `38 100% 96%` — близко, но нужно выровнять.

### 4. Семантика «critical» (красный)
Эталон Red: `#FEF3F2 → #FEE4E2 → #FECDCA → #FDA29B → #F97066 → #F04438 → #D92D20 → #B42318 → #912018 → #7A1202`.

Сейчас:
- `--critical / --destructive: 0 80% 58%` → должно быть **red-500 `#F04438`** ≈ `4 86% 58%` (близко, но точнее под эталон).
- `--critical-soft: 0 80% 96%` → `red-50 #FEF3F2` ≈ `7 86% 97%`.

### 5. Хардкод цветов в `ContributionWidget.tsx` — мимо палитры
В `src/components/oprisk/ContributionWidget.tsx` хардкод вне токенов:
```
hsl(35 92% 52%)   // оранжевый — должен быть orange-500 #FB6514 или amber-500
hsl(232 64% 42%)  // синий — должен быть blue-600 #1570EF (217 90% 51%)
hsl(0 70% 55%)    // красный — red-500 #F04438
hsl(258 60% 55%)  // фиолетовый — purple-500 #9E77ED (258 78% 70%)
hsl(190 70% 45%)  // циан — нет в палитре, заменить на blue-400 #53B1FD или pink-500 #EE46BC (если нужен контраст)
hsl(220 12% 70%)  // серый — gray-300/400
```
Все эти 6 цветов используются как сегменты диаграммы вклада. Нужно перевести на 6-цветный набор из палитры (например: amber-500, blue-600, red-500, purple-500, pink-500, gray-400) и вынести в CSS-переменные `--chart-1 … --chart-6`, чтобы избежать повторного хардкода.

### 6. Градиент AI — вне палитры
`--gradient-ai: linear-gradient(135deg, hsl(200 90% 56%), hsl(170 80% 50%))` — это голубой→бирюзовый, бирюзы в эталонной палитре нет. Варианты:
- A) Перевести на palette-цвета: `blue-500 #2E90FA → green-500 #12B76A` (свежий бренд-градиент, AI = «технология + спокойствие»).
- B) Оставить только blue-палитру: `blue-400 #53B1FD → blue-600 #1570EF`.
- C) Purple→blue: `purple-500 #9E77ED → blue-500 #2E90FA`.

Аналогично `--gradient-ai-soft` (фон AI-карточки) → `blue-50 #EFF8FF → green-50 #ECFDF3` (или соответствующий вариант).

`--gradient-attention: amber-500 → orange-600 #EC4A0A` — почти ок, выровнять под палитру.

### 7. App.css — артефакты Vite-стартера
`src/App.css` содержит `#646cffaa`, `#61dafbaa`, `#888` (логотипы React/Vite). Эти стили не используются дашбордом, но засоряют. Удалить мёртвые блоки или зачистить.

### 8. `src/tailwind.config.lov.json` — параллельная палитра
Этот файл содержит полные шкалы Tailwind по умолчанию (slate/gray/zinc/neutral/stone/red/orange/...). Если он подхватывается сборкой, мы рискуем смешать «нашу» палитру с дефолтной Tailwind. Проверить, нужен ли вообще, и если нет — удалить. Если нужен (для прокси-токенов) — заменить значения шкал на эталонную палитру.
