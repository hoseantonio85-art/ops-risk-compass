## Что меняем

### 1. AISummary.tsx — убрать дубли действий
Удалить нижний ряд кнопок (`К лимиту`, `Проверить меры`, `Почему такой вывод`).

Обоснование: AI-саммари — это **смысл и контекст**, а не место для primary-действий. Действия должны быть привязаны к конкретным виджетам (LimitCards → drawer, FocusCases → карточка кейса) или жить в едином месте (будущий BottomActionBar). Сейчас «К лимиту» дублирует то, что уже выражено визуально через LimitCards и через ссылки внутри блока «Рекомендуем начать с».

Что остаётся в AISummary:
- Заголовок с иконкой Sparkles
- Основной нарратив (situation summary)
- Блок «Рекомендуем начать с» (1, 2)
- Подпись «Это важно, потому что…» — объяснение вывода

То есть блок становится чисто **insight-блоком** без CTA.

### 2. LimitCards.tsx — drill-down через Sheet

Каждая из 4 карточек становится `<button>` (или обёрнута в `SheetTrigger`), при клике открывается правый Drawer с детализацией по выбранной категории потерь.

**Содержимое drawer для карточек «прямые / косвенные / кредитные потери»:**
- Header: название категории + большая цифра (факт / лимит / %)
- Блок «Динамика за 12 месяцев» — `BarChart` (recharts) по месяцам, помесячные потери vs пропорциональный месячный лимит (тонкая референс-линия)
- Блок «Топ инцидентов в категории» — список 5 инцидентов: дата, описание, сумма потерь, статус. Каждая строка — кликабельная (визуально, без перехода — это прототип)
- Footer-CTA: `Открыть реестр инцидентов` (категория уже как фильтр)

**Содержимое drawer для карточки «Прогноз потерь»:**
- Header: прогноз на конец года + дельта к вчера
- Большой sparkline/линия за 30 дней
- Список «Что повлияло на прогноз сегодня» — 3 фактора (например: «новый инцидент в кредитном портфеле +1.4 млн», «пересчёт сценария X +0.7 млн»)
- Footer-CTA: `Открыть модель прогноза`

**Технически:**
- Внутри `LimitCards` хранить state `openCategory: string | null`.
- Обернуть всю секцию в один `<Sheet open={!!openCategory} onOpenChange={…}>` с `SheetContent side="right" className="w-full sm:max-w-xl overflow-y-auto"`.
- Контент drawer рендерится по `openCategory`, моки внутри компонента.
- Сама карточка получает: `role="button"`, `tabIndex={0}`, обработчики click/keyboard, `cursor-pointer`, focus-ring, hover уже есть.

Моки динамики (12 точек на категорию) и топ-инцидентов держим прямо в файле, чтобы не плодить файлы.
